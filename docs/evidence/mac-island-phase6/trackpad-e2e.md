# Phase 6 Trackpad E2E Evidence

## Result

`trackpad-e2e-fallback.json` is generated by `generate-trackpad-e2e-evidence.swift`. It exercises the production trackpad adapter with reducer and motion-plan wiring. Every accepted row records the emitted intent, reducer reason, resolved motion kind, deterministic motion transition ID, final visual state, and any mock track command.

## Covered Flows

- Vertical down: compact -> activity -> expanded.
- Vertical up: expanded -> activity -> compact.
- Horizontal music: previous and next mock tracks.
- Ignored paths: below threshold, cooldown duplicate, reset after 161 ms, and horizontal command in app mode.
- Axis contract: equal axes prefer horizontal; a larger vertical component selects presentation movement.

## Host and Fallback

The native host path is `IslandInteractionHostingView.scrollWheel`, then `IslandWindowController.handleScrollWheel`. A real Mac trackpad must be used to capture AppKit `NSEvent` delivery and the controller's expanded-up hover-focus recovery (`keepsTrackpadHoverFocus` plus interactive hover activation).

This environment has no GUI input capture, so the JSON artifact is explicitly a Swift fallback probe. It validates the same thresholds (70), reset window (160 ms), cooldown (320 ms), reducer behavior, motion profile selection, animation transition IDs, and mock commands. It does not claim real-device gesture delivery, physical trackpad feel, or hover-focus visual capture.

## Manual Capture Procedure

1. Launch the native preview and select a logged-in review compact scenario.
2. Swipe down twice, then swipe up twice; record the compact/activity/expanded/activity/compact sequence.
3. Select music activity. Swipe left and right, confirming previous/next mock titles without a real media provider.
4. From expanded review, swipe up and verify the island remains hover-interactive while recovering activity presentation.
5. Attach the screenshot/video and device/macOS details beside the generated JSON when GUI capture is available.
