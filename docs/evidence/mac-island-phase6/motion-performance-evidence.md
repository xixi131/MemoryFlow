# Phase 6 Motion Performance Evidence

`motion-performance-evidence.json` is generated by `generate-performance-evidence.swift`. It performs deterministic native-model sampling for shell morphing, expanded shadow values, music waveform heights, content choreography, rapid animation-driver retargeting, and an 80-iteration reducer stress sequence.

The stress sequence repeats hover, tap, outside collapse, trackpad, mode-switch, and mock-scenario replacement events. It is intended to expose invalid model samples, stale state convergence, and unbounded per-transition work in the native code path.

## Scope

The generated cadence values use simulated monotonic 60Hz and 120Hz timestamps. They are not monitor measurements. A zero dropped-frame indicator only means every deterministic sample was valid; it does not claim that a physical device sustained a particular FPS or main-thread budget.

The source audit records the bounded clock design: `IslandAnimationDisplayLink` is stopped when idle, the waveform uses a local paused `TimelineView`, and the mock music clock is a pure elapsed-time calculation. Shape construction has no panel invalidation or timer reference. Content/greeting delayed callbacks are epoch-gated to reject stale state.

## Missing Physical Profiling

No physical 120Hz display, Instruments Time Profiler, Core Animation FPS trace, GPU trace, or GUI capture was available for this run. Add those artifacts alongside the JSON before treating frame cadence or main-thread cost as device-performance acceptance.
